## 一、项目概述

### 1. 项目背景
高校学生跨专业交流存在“信息茧房”，传统渠道效率低、信息杂。本项目拟构建轻量级跨专业学习交流平台。

### 2. 项目目标
实现一个最小可行产品，提供需求发布、智能专业匹配、轻量内容社区与站内沟通功能，帮助学生高效寻找跨专业伙伴。

### 3. 开发环境与条件
- 前端：Vue 3 + Element Plus  
- 后端：Spring Boot + MyBatis  
- 数据库：MySQL 8.0  
- 工具：IntelliJ IDEA / VS Code，Git协作  

---

## 二、需求分析

### 1. 目标用户与使用场景
**用户：** 在校本科生、研究生及创新创业团队。  
**场景：** 发布学习需求、寻求跨专业帮助、组建项目团队等。

### 2. 核心功能清单
- 用户注册登录（学号验证/权限分层）  
- 需求发布与专业标签匹配  
- 内容社区（按专业筛选、浏览）  
- 站内即时消息（WebSocket）  
- 管理员后台审核  

### 3. 非功能需求
- 性能：核心接口响应<500ms  
- 安全：密码加密、防注入、Token鉴权  
- 兼容：主流浏览器、响应式布局  

---

## 三、技术路线

### 1. 总体架构
前后端分离，RESTful API + WebSocket。

### 2. 主要技术
- 前端：Vue 3 + Element Plus + Axios  
- 后端：Spring Boot + MyBatis + JWT  
- 数据库：MySQL 8.0  
- 实时通信：WebSocket  

### 3. 数据设计初步考虑
核心实体：用户、帖子、专业标签、消息。  
实现多对多标签关联，支撑按专业筛选与匹配。
